Soru 1 — EBOB ve Öklid Algoritması
a) EBOB(414, 662)
Bulun:Öklid algoritması büyük sayıyı küçüğe bölüp kalanla ilerler:

662 = 1 * 414 + 248
414 = 1 * 248 + 166
248 = 1 * 166 + 82
166 = 2 * 82 + 2
82 = 41 * 2 + 0
Sonuç: Kalan 0 olmadan önceki son pozitif kalan 2'dir. EBOB = 2

b) Pseudocode:
function EBOB(a, b):
while b!= 0:
kalan = a mod b
a = b
b = kalan
return a

c) Zaman Karmaşıklığı:
O(log(min(a, b))). Logaritmik bir büyüklüktür çünkü her adımda sayılar hızla küçülür.
Ekstra:
Döngü yoksa ⇒ O(1)
Bir for döngüsü varsa ve n'e kadar gidiyorsa ⇒ O(n)
İç içe iki for döngüsü varsa ⇒ O(n^2)
Sürekli ikiye bölerek ilerliyorsa ⇒ O(log n)

Soru 14: Faktöriyel
Iterative (Döngüsel): Bir for döngüsüyle 1'den n'e kadar çarpar. Bellek dostudur.
Recursive (Özyinelemeli): Fonksiyonun kendisini çağırmasıdır (n! = n * (n-1)!). Kod daha şıktır ama bellek (stack) harcar.



Soru 15: Kombinasyon Hesaplayan Pseudocode
Pascal Bağıntısı ile (Recursive): C(n,r) = C(n-1, r-1) + C(n-1, r)
FUNCTION Faktoriyel(sayi):
sonuc = 1
FOR i FROM 1 TO sayi:
sonuc = sonuc * i
RETURN sonuc
FUNCTION Kombinasyon_Faktoriyel(n, r):
// n'den küçük r değerleri için hesaplama yapılır
pay = Faktoriyel(n)
payda = Faktoriyel(r) * Faktoriyel(n - r)
RETURN pay / payda



Soru 16 — Modüler Üs Alma (Hızlı Üs Alma)

ALGORİTMA HizliUsAlma(taban, us, mod_degeri)
sonuc = 1
taban = taban MOD mod_degeri

WHILE us > 0 DO
// Eğer üs tek sayı ise, sonucu mevcut taban ile çarp
IF (us MOD 2 == 1) THEN
sonuc = (sonuc * taban) MOD mod_degeri
END IF
// Üssü yarıya indir (Tam bölünür)
us = us / 2
// Tabanın karesini al ve modunu al (Sayıyı küçük tut)
taban = (taban * taban) (MOD mod_degeri)
END WHILE
RETURN sonuc
Neden doğrudan üs almaktan verimlidir?
Cevap: Çünkü doğrudan a^b hesaplamak çok büyük sayılar (overflow) üretir. Hızlı üs alma (Square and Multiply) her adımda mod alarak sayıları küçük tutar ve işlem sayısını O(b)'den O(log b)'ye düşürür.



Soru 17 — Tüm Altkümeleri Üretme (Power Set)
[1, 2, 3] Girişi: Toplam 2^n yani 2^3 = {8} altküme oluşur.
Kombinasyon İlişkisi: Altküme sayısı, aslında {n}{0} + {n}{1} + ... + {n}{n} toplamına eşittir.

ALGORITHM GenerateSubsets(dizi, index, current_subset)
// BASE CASE: Eğer dizinin sonuna geldiysek altkümeyi yazdır
IF index == dizi.length THEN
PRINT current_subset
RETURN
END IF

// 1. OPTION: Mevcut elemanı altkümeye DAHİL ETME (Exclude)
GenerateSubsets(dizi, index + 1, current_subset)

// 2. OPTION: Mevcut elemanı altkümeye DAHİL ET (Include)
current_subset.add(dizi[index])
GenerateSubsets(dizi, index + 1, current_subset)

// BACKTRACKING: Eklediğimiz elemanı geri çıkar
current_subset.remove(dizi[index])

END ALGORITHM

Soru 19 — Sayılar Teorisi (Asal Listeleme)
a) Basit Yöntem: 1'den N'e kadar her sayı için isPrime(n) fonksiyonunu çağırmak.
ALGORITHM ListPrimes(N)
FOR i FROM 2 TO N DO
isPrime = TRUE
// i'nin asal olup olmadığını kontrol et (2'den kök i'ye kadar)
FOR j FROM 2 TO SQRT(i) DO
IF i MOD j == 0 THEN
isPrime = FALSE
BREAK // Bölünen bulduk, döngüden çık
END IF
END FOR

IF isPrime == TRUE THEN
PRINT i
END IF
END FOR
END ALGORITHM


b) Verimli Yöntem: Eratosten Kalburu (Sieve of Eratosthenes). Sayıları silerek ilerler ve çok daha hızlıdır.
ALGORITHM SieveOfEratosthenes(N)
// 1. ADIM: N boyutunda bir BOOLEAN dizi oluştur ve hepsini TRUE yap
// (Başlangıçta her sayıyı asal kabul ediyoruz)
isPrime = ARRAY of size N+1 filled with TRUE

// 0 ve 1 asal değildir, onları FALSE yap
isPrime[0] = FALSE
isPrime[1] = FALSE

// 2. ADIM: 2'den kök N'e kadar olan sayıları kontrol et
FOR p FROM 2 TO SQRT(N) DO
// Eğer p hala asal olarak işaretliyse, onun katlarını ele
IF isPrime[p] == TRUE THEN
// p'nin katlarını (2p, 3p, 4p...) FALSE (asal değil) yap
FOR i FROM p*p TO N STEP p DO
isPrime[i] = FALSE
END FOR
END IF
END FOR
// 3. ADIM: TRUE kalan tüm indisleri yazdır (Bunlar asallardır)
FOR i FROM 2 TO N DO
IF isPrime[i] == TRUE THEN
PRINT i
END IF
END FOR
END ALGORITHM
